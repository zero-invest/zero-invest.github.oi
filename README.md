# 溢价率日常看板

这是一个面向 LOF / QDII 场内溢价率观察的静态网站项目，目标是替代直接拿 T-2 净值估溢价率的粗糙做法。

当前版本已支持 20 只基金的日常看板，并为每只基金维护独立误差记录与独立修正模型。

## 当前能力

- 启动时自动抓取最新基金净值、历史净值和场内价格
- 日度资料缓存到 .cache/fund-sync/daily，避免一天内重复抓取
- 日内场内价格、USD/CNY 和 161128 前十大持仓报价缓存到 .cache/fund-sync/intraday
- 自动记录昨日估值与后续真实净值的差值，并累积误差历史
- 首页表格主看板支持排序，展示现价、估值、溢价率、净值日期、现价时间和限购状态
- 详情页展示每只基金自己的误差折线图和模型状态
- 161128 额外保留持仓级估值实验室，并自动带入 USD/CNY 与美股前十大持仓报价

## 估值方法

首页自动模型当前使用：

自动估值 = 最近官方净值 × (1 + alpha + betaLead × 场内当日涨跌幅)

其中：

- 场内当日涨跌幅 = 当前场内价格 / 昨收价 - 1
- alpha、betaLead 由各基金自己的误差历史单独更新
- 当真实净值发布后，系统会把当日预测误差回写到该基金自己的模型

161128 详情页额外提供持仓模式：

- 以官方 T-1 净值为锚点
- 使用前十大持仓权重和代理篮子做净值贡献拆解
- 叠加汇率变化、费用拖累、人工修正和独立细模型修正

## 本地开发

安装依赖后可直接运行：

```powershell
npm install
npm run dev
```

如果需要自动同步并打开本地页面：

```powershell
npm run app:start
```

如果只需要刷新一次数据：

```powershell
npm run sync:data
```

默认开发地址为 http://localhost:5173。

## 部署

推荐使用 GitHub Pages。

- 网站本体按静态站点发布，不需要访问时保持本地电脑在线
- 数据同步和构建由 GitHub Actions 定时执行
- 别人直接打开网址即可查看最近一次更新后的页面
- 路由使用哈希模式，避免静态托管下的前端路由 404 问题

部署所需工作流文件已包含在 [deploy-pages.yml](.github/workflows/deploy-pages.yml)。
