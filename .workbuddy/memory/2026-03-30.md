## 第二轮修复（17:30 - 17:55）

### 用户反馈
- 截图显示净值列 `--`、限购显示全是"不限购"、溢价率 0.00%
- 用户要求：Cloudflare 网页端不依赖本地脚本，独立运行

### 根本原因（重要！）
1. **前端 App.tsx 数据加载策略错误**：线上非 localhost/GitHub 环境会优先 Worker API，而 Worker 的净值数据全是 0（东财 API 被反爬，境外 IP 封锁），所以用户看到的净值全是 `--`
2. **Worker sync-engine.js 两处 bug**：
   - `purchaseLimit` 暂停申购时返回 `'0元'`（应改为 `'暂停申购'`）
   - `fetchPurchaseStatus` timeout 参数写在 options 对象里不生效
3. **Worker 净值 API 根本问题**：东财 `chart/lsjz`、`f10/lsjz`、`pingzhongdata.js` 在 Cloudflare Worker 境外 IP 环境被反爬封锁，64个基金只有约12个能成功取到净值

### 实际修复内容
1. **App.tsx**：改为所有环境都优先静态文件，Worker API 只作兜底，加净值质量检测（>60%净值为0则跳过）
2. **cloudflare/worker/src/sync-engine.js**：
   - `fetchOfficialNav` 改用 `chart/lsjz?callback=GetData` 为主数据源（三层 fallback）
   - 暂停申购时 `purchaseLimit` 返回 `'暂停申购'`（而不是 `'0元'`）
   - timeout 参数修为第三个参数
3. **部署**：`npm run build:static` + `npx wrangler pages deploy` + Worker `npx wrangler deploy`
4. **手动触发同步**：`POST /internal/sync/runtime?force=true` (token: debug-sync-token)

### 验证结果
- `premium.leo2026.cloud/generated/funds-runtime.json`：64 个基金，净值0个，数据正常
- Worker API 净值仍为 0（东财被封），但前端已 fallback 到静态文件，不再影响用户

### 关键经验
- 同步脚本优先从日缓存读取（当日缓存version=DAILY_CACHE_VERSION时），修改逻辑后需同步更新缓存
- PowerShell 终端显示中文乱码是编码问题，不影响实际数据（用 node 脚本验证）
