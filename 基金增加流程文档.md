# 基金增加流程文档

## 一、流程概述

本流程用于指导如何向LOF溢价率估值网站添加新的基金。整个流程包括准备工作、数据采集、系统配置、云端部署、功能测试和文档记录六个主要步骤。

## 二、详细流程

### 1. 准备工作

#### 1.1 系统架构了解
- **前端**：React 18 + TypeScript，部署在Cloudflare Pages
- **后端**：Cloudflare Workers，使用D1数据库存储数据
- **数据来源**：腾讯行情、新浪汇率、天天基金
- **核心配置文件**：
  - `src/data/fundCatalog.json`：基金基本配置
  - `src/data/fund-short-names.json`：基金名称配置
  - `cloudflare/worker/src/sync-engine.js`：数据同步引擎

#### 1.2 工具准备
- 代码编辑器（VS Code）
- Node.js环境
- Cloudflare Wrangler CLI
- 网络浏览器

### 2. 数据采集

#### 2.1 基本信息采集
- **基金代码**：6位数字代码
- **基金名称**：完整名称和简称
- **基金类型**：LOF、ETF、混合型等
- **风险等级**：低、中、高风险
- **基金公司**：基金管理公司名称
- **成立时间**：基金成立日期
- **管理规模**：最新管理规模

#### 2.2 持仓数据采集
- **前十大持仓**：股票代码、持仓数量、持仓占比
- **行业分布**：主要投资行业
- **资产配置**：股票、债券、现金等比例

#### 2.3 历史业绩采集
- **近1月、3月、6月、1年收益率**
- **今年来收益率**
- **成立以来收益率**
- **同类排名**

#### 2.4 数据来源
- 天天基金网
- 新浪财经
- 基金公司官网

### 3. 系统配置

#### 3.1 修改fundCatalog.json

**文件路径**：`src/data/fundCatalog.json`

**配置字段说明**：
- `code`：基金代码（必填）
- `detailMode`：详情模式，一般为"summary"
- `priority`：优先级，数字越小优先级越高
- `pageCategory`：页面分类，如"domestic-lof"、"qdii-lof"、"etf"
- `estimateMode`：估值模式，"market"或"proxy"
- `proxyBasketKey`：代理篮子键（仅当estimateMode为"proxy"时需要）

**示例**：
```json
{
  "code": "501227",
  "detailMode": "summary",
  "priority": 65,
  "pageCategory": "domestic-lof",
  "estimateMode": "market"
}
```

#### 3.2 修改fund-short-names.json

**文件路径**：`src/data/fund-short-names.json`

**配置字段说明**：
- `code`：基金代码（作为键）
- `shortName`：基金简称（必填）
- `fullName`：基金全称（可选）

**示例**：
```json
{
  "501227": { "shortName": "泓德红利优选" }
}
```

#### 3.3 代理篮子配置（如需）

**文件路径**：`cloudflare/worker/src/sync-engine.js`

**配置字段说明**：
- `key`：代理篮子键
- `benchmark`：基准名称
- `tickers`：代理股票代码数组

**示例**：
```javascript
const PROXY_BASKET_META = {
  'us-oil': { benchmark: '原油', tickers: ['USO', 'BNO'] },
  // 其他代理篮子...
};
```

### 4. 云端部署

#### 4.1 重新部署Cloudflare Worker

**命令**：
```bash
cd cloudflare/worker
npx wrangler deploy
```

**验证**：
- 部署成功后会显示Worker URL
- 确认部署状态为成功

#### 4.2 触发数据同步

**方法**：使用POST请求触发同步

**URL**：`https://<worker-url>/internal/sync/runtime?force=true`

** headers**：
- `Authorization: Bearer <sync-token>`

**验证**：
- 同步响应中`catalogCount`应增加1
- 同步状态为`ok: true`

### 5. 功能测试

#### 5.1 API响应测试

**URL**：`https://<worker-url>/api/runtime/all`

**验证**：
- 状态码：200
- 响应时间：正常
- 数据完整性：包含新基金数据
- `fundCount`：应增加1

#### 5.2 前端展示测试

**方法**：访问网站首页

**验证**：
- 新基金在列表中显示
- 基金名称正确
- 数据更新及时
- 与其他基金展示风格一致

#### 5.3 系统稳定性测试

**验证**：
- 无错误日志
- 其他基金数据正常
- 同步过程顺利

### 6. 文档记录

## 三、技术要点

### 1. 估值模式选择

- **market模式**：适用于国内LOF基金，直接使用市场价格估值
- **proxy模式**：适用于QDII基金，使用代理篮子估值

### 2. 优先级设置

- 优先级数字越小，基金在列表中显示位置越靠前
- 建议根据基金的重要性和流动性设置优先级

### 3. 数据同步

- 系统每分钟自动同步一次
- 手动触发同步可使用`force=true`参数
- 同步过程中会批量获取基金数据

### 4. 时区处理

- 系统已修复UTC+8时区问题
- 数据时间戳显示为中国本地时间

### 5. 错误处理

- 数据获取失败时会使用备用数据源
- 系统会自动处理网络异常和数据异常

## 四、常见问题

### 1. 基金数据不显示

**可能原因**：
- 配置文件修改错误
- Worker未重新部署
- 同步未触发
- 数据获取失败

**解决方案**：
- 检查配置文件格式
- 重新部署Worker
- 手动触发同步
- 检查数据源可用性

### 2. 估值模式选择

**建议**：
- 国内LOF基金：使用market模式
- QDII基金：使用proxy模式，并配置相应的代理篮子

### 3. 代理篮子配置

**注意事项**：
- 代理篮子应选择与基金投资方向相关的股票
- 代理股票应具有良好的流动性
- 权重分配应合理
